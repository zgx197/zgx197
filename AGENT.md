# Repository Agent Guide

这份文档面向在当前仓库中协作的 AI 助手，例如 GPT、Codex 或其他代码代理。目标是帮助 AI 快速理解仓库用途、当前关注重点，以及在修改文档或提交内容时需要遵守的基本约束。

## 仓库定位

当前仓库主要用于记录我正在推进的开源项目、实验方向与阶段性重点。

它更像一个总览入口，而不是某个单一产品的实现仓库。因此，AI 在这里的主要工作通常是:

- 维护 `README.md` 等说明文档
- 更新当前重点项目与项目列表
- 补充协作约定、说明性文档与入口信息
- 保持仓库描述与最近的真实工作重心一致

## 当前更新重点

当前阶段的首要关注对象是 `D:\UGit\Tessera\`。

如果需要更新仓库说明、近期重点或项目描述，应优先与 Tessera 的当前方向保持一致，避免继续把已经降级的项目写成主线。

## AI 协作原则

AI 在修改当前仓库时，应优先遵守以下原则:

- **先读再改**: 在修改 `README.md`、说明文档或仓库约定前，先阅读现有内容，理解当前表述方式与上下文。
- **小步修改**: 优先做最小、明确、可解释的改动，不要无故重写整份文档。
- **保持一致**: 如果更新了“当前重点”，要检查相关描述是否也需要同步调整，避免前后矛盾。
- **尊重既有内容**: 不要删除尚未明确废弃的项目记录，除非任务明确要求移除。
- **中文优先**: 面向仓库说明、协作约定和提交信息时，默认使用中文表达；必要时可保留英文项目名、路径和技术术语。
- **不要臆造状态**: 如果某个项目的阶段、路线图或规则无法从仓库内容确认，应使用保守表述，不要编造已完成或已决定的事项。

## 文本文件规范

当前仓库中的文本文件默认应遵守以下格式要求:

- 使用 `UTF-8` 编码
- 使用 `LF` 换行
- 不要写入 `CRLF`
- Markdown 文档默认使用普通文本结构，避免无意义的复杂格式

仓库中的 [`.gitattributes`](./.gitattributes) 当前已声明:

```text
* text=auto eol=lf
```

AI 在新增或编辑文档时，应主动保持这一约束，不要引入与仓库规范不一致的换行格式。

## 提交信息规范

当前仓库的提交说明应使用“前缀 + 中文描述”的格式:

```text
prefix: 中文说明
```

例如:

- `docs: 更新当前重点为 Tessera`
- `chore: 补充 AI 协作说明`
- `fix: 修正文档中的错误链接`

当前 hooks 校验的前缀包括:

- `feat`
- `fix`
- `docs`
- `style`
- `refactor`
- `perf`
- `test`
- `build`
- `ci`
- `chore`
- `revert`

说明:

- 提交主题中必须包含中文描述。
- 当前仓库的 hook 按 `prefix: 中文说明` 进行校验。
- 如果提交说明不符合规则，提交或推送可能被拦截。

相关脚本位于:

- [`.githooks/validate-commit-message.ps1`](./.githooks/validate-commit-message.ps1)
- [`.githooks/validate-pre-push.ps1`](./.githooks/validate-pre-push.ps1)

## 对 AI 的额外建议

- 修改 README 时，优先保证“当前重点”“项目状态”“仓库定位”三处信息互相一致。
- 如果需要引用外部仓库，优先使用已经确认存在的本地路径或仓库地址。
- 如果发现仓库规则与实际脚本校验不一致，应优先按实际校验规则表述。
- 除非任务明确要求，否则不要擅自扩展为复杂流程文档，先保持约定清晰、简短、可执行。

## 关于这里

我更希望把代码仓库做成适合长期维护、方便阅读、也方便他人参与的样子：

- 有清晰的问题边界
- 有可复现的使用方式
- 有持续补充的文档说明
- 有可以逐步迭代的工程结构

## 我关注的方向

- 实用型开源工具
- 自动化与效率提升
- 工程化实践与开发体验
- 更易读、更易维护、更容易协作的项目设计

## Roadmap

围绕 AI 工具、游戏开发与工程化工作流持续迭代，按项目逐步推进近期目标、文档和可复用能力。

### 枚举说明

| 维度 | 枚举值 | 含义 |
| --- | --- | --- |
| 项目状态 | `迭代中` / `探索中` / `整理中` / `暂停中` | 表示项目所处阶段，用来描述项目生命周期，而不是当前是否优先投入。 |
| 方向 | `AI 工具链` / `游戏与玩法` / `游戏与运行时` / `设计到交付` / `工具链` | 表示项目所属的问题域，用来帮助快速理解项目类型。 |

## 开源协作方式

如果你对某个项目感兴趣，欢迎通过下面的方式参与交流：

- 提交 Issue 反馈问题或提出建议
- 提交 Pull Request 一起完善实现
- 在项目页面留下 Star，帮助我判断哪些方向值得继续投入
